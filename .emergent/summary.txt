<product_requirements>
The application aims to be a full-stack production kitchen management system.
**Core Features & Iterative Enhancements:**
*   **Production Management:** Initially allowed adding/confirming items; evolved to include categories, units of measure, auto-appearing items, and automatic 15% markup. The 'produced' column was removed, and quantity limitations were lifted, meaning all items are always available in 'kg' units. Managers can now create, edit, and force-delete production items, with data integrity protected against order references.
*   **Venue Ordering:** Started with manager-set quantities. Now features visual displays, category tabs, previously/most ordered sections, and quantity pickers. All items are always orderable without kitchen completion status dependency.
*   **Financials:** Includes 15% markup (hidden from non-managers), PO/invoice generation (with 8% tax), and daily revenue tracking. Invoices are Xero-compatible with PDF export (invoice number, date, items, ordering venue).
*   **User Management & Access Control:** Predefined roles (manager, kitchen staff, venue staff). Managers can add, edit, and delete users. The system is transitioning to password-based login for all roles, with managers controlling passwords.
*   **Manager Dashboard & Reporting:** Centralized dashboard with tabs for Production, Orders, Users, Invoices, Purchase Orders, and Settings. Includes real-time order notifications with action required alerts.
*   **UI/UX:** Utilizes Tailwind CSS. Features include location address option, delivery date section, image uploads, manager-customizable display (colors, fonts, layout). A global Save All Changes button was implemented for manager profile modifications, with visual indicators for unsaved changes.
*   **Notification System:** A comprehensive system for order events (placed, preparing, ready, delivered) is implemented for in-app notifications on kitchen and manager dashboards. Contact management (email, phone) is in place, ready for future email/SMS service integration.
</product_requirements>
